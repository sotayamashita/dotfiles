# 読者プロファイル(雛形)

スキルはプロファイルの実体を持たない。実体は環境が与える —
プロジェクトの AGENTS.md が指すファイル、またはカレントディレクトリの
`PROFILE.md`(SKILL.md 工程 3 の解決順)。どちらも無いときは、この雛形の
見出しに沿って読者に 2〜3 問聞き取り、カレントディレクトリに `PROFILE.md`
を作成してから生成を続行する(個人データなのでリポジトリへのコミットに注意)。

前提チェックの突き合わせ先、および「現実世界で回してみる」の題材源。
読者の現場が変わったら実体ファイルを編集する。

## 背景知識

〔読者の職種と、前提にしてよい知識・してはいけない知識を書く。
数式の扱い(記法を前提にできるか、日本語の読み下しが要るか)を必ず含める。〕

## 現場(現実例はここに寄せる)

〔読者の日常業務と、例に使ってよい具体物(担当サービス、ツール、
運用しているもの)を列挙する。〕

## 読み方の好み

〔言語、読了目安、用語表記(原語併記の要否)、説明の好み。〕
