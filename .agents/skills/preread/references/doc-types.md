# 文書タイプ別の適応

取得した本文を読み、まずタイプを 1 つ判定する。迷ったら「著者が何を
根拠に主張しているか」で切り分ける(実験データ → 実証、経験と見立て →
立場表明かブログ、他者の仕事の整理 → サーベイ、手順 → チュートリアル)。

タイプは排他ではなく混成しうる(例: 技術ブログでありながら立場表明)。
その場合は主となるタイプの骨格で心像図を組み、もう一方の規約(色分けなど)を
重ねて適用する。kicker には混成のまま書いてよい(例: 「立場表明(技術ブログ)」)。

## 心像図の骨格(タイプごとに切り替える)

心像図のノード構成はタイプで決まる。以下は初期テンプレート
(運用で誤りが見つかったら、ここを更新する):

- **実証論文**: 問い → 手法 → 証拠 → 限界。
  証拠ノードには主要な実験結果の数字を 1 つ入れる
- **立場表明(ポジションペーパー)**: 中心の問い → 主張 → 柱(複数) →
  実装・限界。柱が主張をどう支えるかを結線(WIRES)で見せる
- **サーベイ**: 分類の地図。中心テーマ → 分類軸 → 各系統 →
  著者が見る今後の方向。系統間の対立や補完を結線で見せる
- **チュートリアル**: 段階の一本道。前提 → 手順の節目 → 完成形 →
  つまずきどころ
- **ブログ**: 著者の問い → 事実(確立された知見) → 著者の見立て →
  読者への含意。事実と見立てをノードの段階で分ける

## ブログ・立場表明での「事実」と「見立て」の峻別

- 確立された事実と著者の意見を、本文中で色分けして峻別する:
  著者の見立ては朱の囲みで明示する(「著者の見立て」ラベル付き)
- 種別ラベル(kicker・meta)にも検証の程度を明記する
  (例: 「立場表明(実験による検証は限定的)」)

## 既読との接続

- 過去に生成した教材と主題が重なるときは、「既読との接続」の緑の囲みで
  対応関係を示す(例: 「この節の X は、既読の Y 教材で読んだ Z と同じ
  問題を別の層で解いている」)
- 既読の探索はカレントディレクトリの `preread/` に閉じない。そこに
  無ければ、読者に「主題の重なる既読の文書・教材はあるか」を 1 問だけ
  確認する。preread の出力でなくても(過去のモック教材や読了済みの
  原典でも)主題が重なれば接続してよい

## 取得層の注意

- 取得は arXiv 固有にしない。任意 URL を defuddle で、arXiv は HTML 版
  優先で取る
- ページ分割された連載や、付録が別ページの文書は、判定に必要な範囲まで
  取得してから書き始める
