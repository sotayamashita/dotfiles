# Stop Slop JA

日本語の文章から、slop と呼ばれる LLM 特有の重さを取り除く。人間らしく見せる演出はしない。読み手が一度で掴める文にする。

## 適用手順

書くとき: 内容選択(規則28-30)で主経路を決める。構造(21-27)で段落を設計する。語・文(1-20)を守って書く。渡す前に規則1〜30を上から順に照合する。

推敲するとき:

1. 内容選択(28-30)で残す段落を決める。消す文は磨かない。
2. 構造(21-27)で段落の順序と分割を直す。
3. 機械的に判定できる語・文規則(17-20)を一括で当てる。
4. 判断を要する語・文規則(1-16)を通読で直す。
5. 規則1〜30を上から順に照合してから渡す。

## 語・文レベルの規則

一文で判定できる。詳細と Bad/Good の例は [references/words.md](references/words.md)(規則と同じ番号)。

1. 動詞で書く。「精度低下」ではなく「精度が落ちる」。誰が何をするかを見せる。
2. 一文一義。「〜で、」「〜が、」「〜ので、」で命題を繋がない。
3. 機能語で呼吸を入れる。内容語を詰めた電報文にしない。
4. 挿入補足は削除テストにかける。括弧を消して意味が変わらないなら消す。削れない補足は独立した文にする。
5. 和語を選ぶ。「毀損する」ではなく「削られる」。
6. 英語の専門用語は動作に書き下す。「calibrate する」ではなく「人間の採点結果に合わせて基準を調整する」。
7. 固有名詞は原語のまま残す。定義した側(企業、コミュニティ、仕様)が名前として扱う語は訳さない。
8. カタカナ外来語は日本語で言い換える。定着した語(バス、テレビ)は使ってよい。
9. 実際に何が起きるかで書く。「本番品質の」ではなく「顧客に出せる」。
10. どの文章にも置ける語(「重要な」「包括的な」)を消し、その文章に固有の語を使う。
11. 行動の主体を主語にする。「検証が行われます」ではなく「システムが検証します」。例外は、旧情報を文頭に置くための受身。
12. 同じ語を近くで繰り返さない。2回目以降は「これ」で受けるか主語を省く。例外は、対比や曖昧さ回避に必要な反復。
13. 同一の対象は同じ語で呼ぶ。言い換えで変化を付けない(規則12の対)。
14. 同じ構文・同じ文長を3回続けない。太字は初出の定義と論理の要所に限り、一節に1〜2箇所まで。
15. 根拠のない保険を積まない。未検証なら「未検証だが」と理由ごと書く。出典は名指しする。逆に、推定・読者の疑念・反実仮想の推量は機械的に断定へ変えない。
16. 誇張しない。「完全に」「革命的な」は事実が支える範囲まで弱める。
17. 二重否定を使わない。「できないわけではない」ではなく「できる」。
18. 意味を足さない語を消す。「することができます」ではなく「できます」。
19. コロンの前は名詞で終える。「実行します:」ではなく「例:」。
20. 組版記号(`“”` `—` など)を持ち込まない。ASCII か日本語の記号を使う。

## 構造レベルの規則

前後の文脈で判定する。詳細と例は [references/structure.md](references/structure.md)(規則と同じ番号)。

21. 説明を先、詳細を後。箇条書きの前に、何の列挙かを名詞終わりの一文で書く。
22. 1段落に主張は1つ。密な3段落より疎な7段落。
23. 文末に核心を置く。但し書きや譲歩で文や段落を終えない。
24. 段落内で話題を揃える。各文の文頭を、段落の話題の系列にする。
25. 接続詞は論理関係があるときだけ置く。実在する関係は接続表現で明示し、「また」「さらに」を水増しに使わない。
26. まとめ行を置かない。必要なまとめは最後の項目の末尾か、次の段落の冒頭へ移す。「本稿では」も消す。
27. 見出しは、その節が答える問いか扱う対象を指す句にする。種別だけ・手順だけの見出しにせず、結論を言い切るセリフにもしない。

## 内容選択レベルの規則

書く前の取捨選択。文も段落も規則を守っているのに全体が重いとき、この層を疑う。

28. 主経路には結論・理由・最小構成・設計ルールだけを置く。全列挙・概念表・FAQ は参照領域(付録、別セクション)へ圧縮する。
29. あとで参照しない固有名・数値・時刻(ファイル名、HTTP ステータス、カバレッジ率)を出さない。「金額計算のユーティリティ」のような一般的な言い方で受け、議論の問い・帰結に効く具体だけ残す。
30. 同じ説明を文書内で繰り返さない。一度書いたら参照する。

- Bad: 本文、概念表、FAQ が同じ概念差分を3回説明する
- Good: 本文に結論と理由。概念表は1行要約、FAQ は本文にない疑問だけ
